## Опис проєкту

Проєкт демонструє спрощену серверну частину маркетплейсу вживаних авто. Система дозволяє отримувати список оголошень про продаж автомобiлiв, переглядати окреме оголошення та створювати заявку на тест-драйв.

Об'єкт iндивiдуального завдання: маркетплейс вживаних авто.

## Глобальнi змiннi середовища

Конфiгурацiя контейнера виконується через змiннi середовища:

| Змiнна | Призначення |
| --- | --- |
| APP_ENV | режим роботи системи |
| APP_PORT | порт запуску контейнера |
| STUDENT_ID | номер студента |
| SERVICE_NAME | назва сервiсу |
| DEFAULT_CURRENCY | валюта за замовчуванням |

Файл `.env.example` мiстить приклад налаштувань без секретних даних.

## Швидкий старт

Скопiювати приклад змiнних середовища:

    Copy-Item .env.example .env

Запустити контейнер:

    docker compose up --build

Перевiрити API:

    Invoke-RestMethod http://localhost:8011/
    Invoke-RestMethod http://localhost:8011/cars
    Invoke-RestMethod http://localhost:8011/cars/1101

## API

GET `/`  перевiрка роботи сервiсу.

GET `/cars`  отримання списку оголошень.

GET `/cars/{car_id}`  отримання конкретного оголошення.

POST `/test-drives`  створення заявки на тест-драйв.

Приклад тiла POST-запиту:

    {
      "car_id": 1101,
      "buyer_name": "Test Buyer",
      "preferred_date": "2026-06-25"
    }

## Автоматизоване тестування

Для перевiрки API використано `pytest`. Тести перевiряють:
- наявнiсть `student_id = 11`;
- коректнiсть ID оголошень;
- створення заявки на тест-драйв.

GitHub Actions автоматично запускає тести пiсля push або pull request.
